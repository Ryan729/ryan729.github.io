使用Docker自建Tailscale实现虚拟内网

发表于 2025-01-27 分类于 工具

一、背景

父母经常有些电脑上的功能不会操作，需要我远程协助，以前使用的是Teamviewer，这几年限制越来越多，账号登陆都成了问题。最近发现了Tailscale和Zerotier等构建虚拟内网的工具，于是决定自建Tailscale + Derper中继节点 + 微软RDP远程桌面的方式来实现远程操作。顺便自己也体验体验，看看有没有更多玩法。

（PS：使用RDP远程登陆，会导致被控端锁屏，不是一个适合的协助父母远程操作的方式）

二、简介

1. Tailscale

Tailscale属于一种虚拟组网工具，基于WireGuard。他能帮助我们把安装了Tailscale服务的机器，都放到同一个局域网。也就是我在家里的NAS和PC，还有父母家的PC，甚至云服务器都能放到同一个局域网。Tailscale几乎支持全平台，各个平台都有对应的客户端。在Linux平台上也有对应的CLI工具，使用起来非常方便。

2. Derper中继服务器

Tailscale会尝试在两个设备间进行STUN打洞，如果打洞成功，会进行直连。如果打洞失败，则要使用中继服务器进行流量中转，在Tailscale体系中，中继服务器被称为Derper。Tailscale的中继服务器节点大多在国外，所以为了更好的使用体验，可以自己构建Derper中继服务。

3. Headscale

https://github.com/juanfont/headscale

Headscale是开源版本的Tailscale控制器，可以用来负责管理设备。但是缺少Web页面。

4. Headscale-UI

https://github.com/gurucomputing/headscale-ui

开源的Headscale Web页面，可以用来管理Headscale设备。

三、环境准备

1. 国内云服务器

最好是ipv4+ipv6双栈，双栈环境很多时候会提高打洞的成功率。

在云服务器厂商面板或者防火墙开启443/tcp和3478/udp端口。

2. 国内备案域名（教程中以demo.com为例）

国内机器的延迟当然最好，但是使用国内服务器就需要备案的域名。

Tailscale Linux CLI工具在使用时会尝试使用HTTPS，所以不使用域名的场景下，经常会导致无法登陆。

所以建议使用域名配置HTTPS进行访问，如果没有备案域名，可以尝试Headscale搭建在国外服务器，Derper中继服务使用国内服务器通过ip搭建访问。本教程为了简化部署流程，使用Headscale内置的Derper中继服务，所以需要使用国内机器配合已备案域名。

3. Docker

使用Docker搭建会很方便很多。

多种方式可以查看设备状态：

  • 在tailscale的客户端中查看设备状态
  • 在Linux设备使用tailscale status查看设备状态，使用tailscale netcheck检测网络状态

五、注意事项

1. headscale如果要自己搭建，建议使用域名上https，因为tailscale客户端有时候需要重连，会默认请求https，如果用ip的话，这时候会连接失败。

2. 客户端如果不需要tailscale的dns相关功能，建议启动时候增加参数–accept-dns=false，或者图形化界面中关闭使用tailscale的dns。否则会覆盖本地的dns，导致一些问题，比如一些国内服务器的内网源解析异常等等。
